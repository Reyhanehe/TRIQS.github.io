Task file generation¶

Documentation:¶

selfconsistency.task_file_generation.generate_files(mydir, params, nprocs, param1, param2, submit='yes', prefix='Sn', queue='Intel', suffix='', deps_dict={}, cluster='weiss')¶

Generate files (python and bash) for cluster computation for ONE point of the phase diagram

Parameters:submit – [yes|no|only]
selfconsistency.task_file_generation.launch_runs(prefix, param1, param1s, param10s, param2, param2s, param20s, schemes, scheme0s, n_loops=20, submit='no', queue='Intel', mydir='./', mydir0='./', n_cycles=200000000000000, nprocs=8, suffix='', max_time=500, cluster='weiss', **kwargs)¶

function that generates and submits runs for ‘param2 VS param1’ diagrams

Parameters:
  • prefix – a prefix that defines the kinetic and interaction parameters used for the calculations [see templates/materials_params.py]
  • param1 – name of the x-parameter [among U|Vnnn|filling|T]
  • param1s – list of param1 to be computed
  • param10s – list of starting param1 for each param1 in param1s [‘n’ for no starting point]
  • param2 – name of the y-parameter [among U|Vnnn|filling|T]
  • param2s – list of param2 to be computed
  • param20s – list of starting param2 for each param2 in param2s [‘n’ for no starting point]
  • schemes – for each param2 in param2s, define the scheme [among edmft|gw_edmft]
  • scheme0s – for each scheme in schemes, define the starting scheme [among edmft|gw_dmft]
  • n_loops – number of self-consistent cycles
  • mydir – directory where to launch calculation
  • mydir0 – directory for starting point
  • n_cycles – number of Monte-Carlo cycles for each self-consistent cycle
  • max_time – maximal time for each Monte-Carlo run
  • submit – [yes|no|only]
  • cluster – cluster name [weiss|kondo]
  • queue – cluster queue [Intel|AMD/standard|test...]
  • nprocs – number of procs per run
